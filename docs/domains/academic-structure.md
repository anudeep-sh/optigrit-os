# Academic Structure Domain

## Purpose

Academic Structure defines how an institution organizes education.

It represents the hierarchy within which teaching occurs.

---

# Core Concepts

Program

Examples:

B.Tech

MBA

MCA

---

Batch

Represents an admission cohort.

Example:

2025 Intake

2026 Intake

---

Semester

Represents a time-bound academic period.

---

Section

Represents a group of students.

Examples:

A

B

C

---

Subsection

Optional subdivision for labs, tutorials or electives.

---

# Responsibilities

Maintain academic hierarchy.

Assign students.

Assign faculty.

Provide context for planning.

Support curriculum organization.

---

# Design Rules

Academic Structure should not contain educational content.

It provides organization, not learning.

Courses belong to Knowledge.

Teaching belongs to Planning.

Attendance belongs to Teaching.

This separation prevents unnecessary coupling.

---

# Why This Exists

Academic structures vary significantly between institutions.

By isolating this domain, OptiGrit can support universities, schools, coaching centers and corporate training without changing the rest of the platform.
