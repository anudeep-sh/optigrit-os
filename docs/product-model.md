# The OptiGrit Product Model

OptiGrit is built around four fundamental layers.

Understanding these layers is essential to understanding the product.

---

# Layer 1

Knowledge

Knowledge represents reusable educational information.

Examples

Subjects

Courses

Learning Assets

Assignments

Resources

Question Banks

Knowledge is reusable.

Knowledge evolves.

Knowledge belongs to institutions.

---

# Layer 2

Planning

Planning determines how knowledge will be delivered.

Examples

Academic Calendar

Teaching Schedule

Lesson Plan

Semester

Section

Faculty Allocation

Planning is contextual.

Planning changes every semester.

---

# Layer 3

Delivery

Delivery represents educational interactions.

Examples

Lecture

Attendance

Discussion

Announcements

Assignments

Assessments

Meetings

Delivery happens in time.

Delivery cannot exist without context.

---

# Layer 4

Outcomes

Outcomes measure educational progress.

Examples

Student Progress

Assessment Results

Attendance Trends

Learning Analytics

Faculty Analytics

Recommendations

AI Insights

Outcomes drive continuous improvement.

# Why This Model Exists

Traditional LMS products organize software around modules.

OptiGrit organizes software around educational workflows.

This distinction affects every architectural decision.

Modules change.

Educational workflows do not.
