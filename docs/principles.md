# Product Principles

Unlike the Constitution, these principles may evolve over time as the platform matures.

---

## Design Around Human Behaviour

Software should adapt to existing educational behaviour whenever possible.

Teachers should not need to learn entirely new ways of working.

---

## Context Over Screens

Users think in terms of context.

A lecture.

A class.

A semester.

A student.

Not modules.

Interfaces should preserve context instead of forcing navigation between isolated screens.

---

## Intelligence Over Configuration

Whenever the platform already knows enough information,

it should make intelligent decisions.

Configuration should become the exception,

not the default.

---

## Progressive Automation

Automation should happen gradually.

Observe.

Recommend.

Assist.

Automate.

Trust is built step by step.

---

## Opinionated Workflows

Too much flexibility creates confusion.

OptiGrit should provide sensible defaults based on educational best practices.

---

## Institutional Memory

Knowledge should accumulate over years.

Teachers come and go.

Institutions remain.

The platform should preserve educational knowledge across generations of educators.

---

## AI as a Teaching Assistant

AI should behave like an exceptional teaching assistant.

It prepares.

Organizes.

Summarizes.

Suggests.

Never replaces the educator.

---

## Build Once, Improve Forever

Every reusable capability should improve over time.

The platform should become more valuable with every semester,

not more complicated.
